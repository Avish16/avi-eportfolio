Project descriptions

Written for readers who may not work in data science every day, with focus on problem, approach, metrics, and outcome.

MediExplain: human-centered medical AI

Stack: LLMs, RAG, ChromaDB, Streamlit · GitHub repository

MediExplain addresses a real communication gap in healthcare. Patients and caregivers often receive technical medical documents that are difficult to interpret. I built a modular system that generates clear, audience-specific explanations while preserving clinical context.

The project uses retrieval-augmented generation with ChromaDB to ground responses in a curated medical corpus. The architecture includes multiple specialized LLM components for routing, explanation, and validation. I added reliability controls such as JSON schema validation, retry logic, and consistency checks, then exposed the workflow through Streamlit for transparent user interaction.

Driver drowsiness detection (deep learning)

Stack: Python, CNN, OpenCV, LSTM · GitHub repository

This project frames drowsiness detection as an eye-state classification problem using infrared-style eye images and driving video. Based on the repository pipeline, the main training workflow uses grayscale 64x64 inputs and deep learning models in TensorFlow/Keras, with data augmentation and held-out evaluation.

The model reaches strong benchmark performance on the curated test split, with approximately 98 percent test accuracy reported in the project results. For deployment, I built an inference pipeline using facial landmarks, left-right eye probability fusion, and time-window aggregation so the system can separate sustained eye closure from normal blinking. OpenCV overlays support both live webcam and offline monitoring.

DataThon: Syracuse code violations and spatial grid analytics

Stack: Python, GeoPandas, Folium, scikit-learn · GitHub repository · Live map dashboard

In this team datathon project, we linked city code violations with the property assessment roll and built a grid-based geospatial workflow for Syracuse. The goal was to identify where risk and vacancy patterns are concentrated and present findings in a way that city stakeholders can explore.

Using the public repository materials, the workflow combines parcel-level joins, feature engineering, and map outputs in Folium. The project includes interactive outputs such as risk maps and prediction dashboards, with emphasis on reproducible processing and practical interpretation instead of opaque scoring.

Bitcoin flash crash prediction

Stack: Jupyter, Python · GitHub repository

This project predicts binary flash crash events in minute-level Bitcoin data by combining market features with time-aligned Reddit sentiment. The modeling notebook builds 60-minute sequences, applies feature scaling, and trains a PyTorch sequence model with validation and early stopping.

Evaluation in the repository reports about 0.71 test accuracy at tuned threshold settings, along with class-wise precision, recall, and confusion matrix analysis. The project demonstrates practical event modeling under class imbalance and highlights how social sentiment can be integrated with high-frequency market features for risk monitoring.
