Project descriptions

Written for readers who may not work in data science day to day—focused on problem, approach, and what I learned.

MediExplain — human-centered medical AI

Stack: LLMs, RAG, ChromaDB, Streamlit · GitHub repository

Many patients and caregivers receive long, technical medical documents. MediExplain is a system that turns that complexity into structured, adaptive explanations. I designed a modular multi-agent setup with many specialized components (routing, validation, and domain-specific “bots”) so answers stay organized instead of becoming a single opaque block of text.

At the core is retrieval-augmented generation: relevant passages are pulled from a curated medical corpus stored in ChromaDB, and the model grounds its explanations in that context. I also built explicit safety and validation layers—JSON validation, retries, and consistency checks—because healthcare-adjacent tools need predictable behavior, not just fluent language. A Streamlit front end makes the workflow transparent and keeps the experience approachable for non-technical users.

Driver drowsiness detection (deep learning)

Stack: Python, CNN, OpenCV, LSTM · GitHub repository

Drowsy driving is dangerous because it is subtle: brief eye closures can be normal blinks, while sustained closure is a different signal. I built a system that works from infrared-style eye images and video, classifies eye state with a convolutional network trained on a large labeled set (85K+ images in the project narrative), and then aggregates over time so the model distinguishes a long closure from ordinary blinking.

The pipeline extracts eye regions using facial landmarks, normalizes inputs, fuses left/right eye predictions, and runs inference with OpenCV overlays. I optimized for real-time use with frame skipping and threshold tuning to balance detection reliability against false alarms. The same ideas support both live webcam demos and offline video review.

DataThon — Syracuse code violations & spatial grid analytics

Stack: Python, GeoPandas, Folium, scikit-learn · GitHub repository

In this datathon project, our team connected city code violations to the property assessment roll and built a spatial grid over Syracuse. The goal is practical: understand where violations cluster, how they relate to assessment and neighborhood context, and communicate results on maps stakeholders can explore.

My contributions align with the public write-up in the repo: joining data on parcel identifiers, engineering grid-level features, and helping produce interactive Folium maps—for example risk surfaces, prediction dashboards, and vacancy-related views. The emphasis is reproducible analysis and clear visuals rather than a black-box score nobody can interpret.

Bitcoin flash crash prediction

Stack: Jupyter, Python · GitHub repository

Cryptocurrency markets exhibit extreme movements that are hard to model with off-the-shelf assumptions. This repository captures my notebook-driven workflow: exploratory analysis, feature thinking, and experiments around flash-crash-style outcomes. It is a strong example of how I iterate in a research notebook—documenting steps so results can be reviewed—and how I connect quantitative metrics to the economic intuition that non-experts care about (risk, tail events, stability of predictions).
